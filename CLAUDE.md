## 1. 何を作るか
四国八十八ヶ所を巡る、**物理巡礼の同行アプリ**。アプリは記録と連れであって、参拝の代わりではない。Ataraxia Works の一員。心の平静＝コンセプト。

## 2. 絶対不変条件（破ったらやり直し）

### 2-1. 宗教的配慮（真言宗・最優先）
- **聖なる固有名を捏造しない。** 札所名・本尊・座標は **四国八十八ヶ所霊場会の公式リスト**等の正典からのみ。データが無い項目は「未入力」と明示し、**推測で埋めない**。
- **朱印・御影を複製/模倣しない。** 参拝のしるしは「納」の一字など、明らかに私的な記号にとどめる。実物の御朱印に似せない。
- **功徳を授けない。** アプリが御利益・功徳を与える表現を出さない。功徳は実際の参拝の行いにある。
- **作法を教えない（僭越回避）。** 参拝作法は各寺・霊場会の案内に従うよう促すのみ。
- `ことわり`（§8）を必ず設置。

### 2-2. 反ゲーム化
- 連続日数・順位・他者比較・SNS・「あと◯ヶ所」「達成率」煽り、すべて**無し**。
- 進行は%でなく**四つの道場**（発心→修行→菩提→涅槃）。結願は静かに。

### 2-3. プライバシー
- 完全オンデバイス。送信・ログイン・広告・追跡・サーバー無し。位置情報を外に出さない。

### 2-4. 可読性（今日の失敗を繰り返さない）
- **文字を薄くしない。** 暗背景なら本文はほぼ白で高コントラスト。屋外で読める前提。
- **細い明朝を本文に多用しない。** 見出しに明朝を使うなら太字。一覧・本文は太めのゴシック。
- `prefers-reduced-motion` 尊重、フォーカス可視、モバイル必須。

## 3. v1 スコープ（今日出す）/ 後回し
**v1（今日）**
- 八十八札所の一覧（道場ごと）
- **自己申告の参拝**（GPS座標に依存しない。タップで「参拝した」）＋一呼吸の間＋一言記帳（任意）
- デジタル納経帳（静かに満ちる）／四道場の進行／結願（お礼参り＝高野山奥の院 を案内）
- すべて端末内に保存（localStorage で可）

**後回し（v1.1+）**
- GPS自動検知・地図・Near-me・全国写し霊場・取り込みパイプライン（sando-design.md §7-8 参照）

## 4. スタック・初期化
- React + Vite + TypeScript + PWA（オフライン対応）。または単一静的HTMLでも可（速度優先）。
- 永続化：v1 は localStorage で十分（~88レコード）。将来 Dexie。
- デプロイ：Cloudflare Pages（静的）。バックエンド無し。

## 5. 札所データ
- `番号(1..88)` と `道場` は確定：**1-23 発心 / 24-39 修行 / 40-65 菩提 / 66-88 涅槃**（自動算出）。
- 寺名は `NAMES: {1:"…", …, 88:"…"}`、本尊は任意 `HONZON: {…}`。**開発者（人間）が公式リストから流し込む**。Claude Code は空欄を捏造しない。
- 未入力の札所は「第N番（札所名 未入力）」と淡くなく**明示**する。

## 6. 参拝フロー（自己申告）
札所をタップ → その寺を示す（番号・道場・あれば本尊）→「一呼吸、おく」の間 → 一言記帳（任意）→「参拝した」で納経帳にひとつ満ちる。演出は静か、音なし。取り消し可。**正直さはユーザーに委ねる（不正検知しない）。**

## 7. UI / トーン
- 静かな黒白基調＋ひかえめな金茶アクセント。ただし §2-4（高コントラスト）を最優先。
- 同行二人＝「ひとりではなく」を一貫。連れであって採点者でない。
- 1画面で巡礼の現在地が分かる（今の道場・納経の満ち具合）。

## 8. ことわり（この文言を設置）
> これは個人の巡礼の記録です。実際の参拝・納経・御朱印に代わるものではありません。寺院の朱印や御影を複製・模倣せず、ここでの「参拝した」は、あなた自身の私的なしるしです。功徳は実際にお参りした行いにあり、このアプリが授けるものではありません。札所・本尊・作法は四国八十八ヶ所霊場会と各寺院に拠ります。参拝の作法は各寺・霊場会のご案内に従ってください。記録はすべて、あなたの端末の中だけに。

## 9. 受け入れ基準（v1 完了条件）

## 10. 働き方（オートパイロット）
- 原則質問せず進む。判断は本書の既定で。独自判断は `DECISIONS.md` に1行記録。
